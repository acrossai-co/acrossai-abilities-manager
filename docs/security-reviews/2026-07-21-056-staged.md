## Vulnerability Findings

### [LOW] SEC-006 — Null-response guard on `bulkSetUserAccessRule` (REMEDIATED)

**Location**: `src/js/abilities/store/index.js` — `bulkSetUserAccessRule`
**OWASP Category**: A04:2025 — Insecure Design
**CWE**: CWE-754 — Improper Check for Unusual or Exceptional Conditions
**CVSS v3.1**: 2.5 (AV:N / AC:H / PR:H / UI:R / S:U / C:N / I:L / A:N)
**Spec-Kit Task**: TASK-SEC-006 (closed in this commit)

**Description**: Initial staged shape relied solely on `apiFetch` throwing on non-2xx. Composer PUT resolving with `null` on a soft failure would give the operator a false-success signal (per `BUG-AC-NULL-RETURN-SILENT-FAIL`).

**Remediation applied**:
```js
const results = await Promise.all(
    slugs.map((slug) => api.setAccessControlRule(slug, acKey, acOptions))
);
const failedCount = results.filter(
    (r) => null === r || undefined === r
).length;
if (failedCount > 0) {
    throw new Error(
        `${failedCount} of ${slugs.length} User Access rules did not save.`
    );
}
dispatch(actions.fetchAbilities());
```

**Status**: ✅ Fixed in the same staged commit. Paired Jest coverage below.

### Durable Memory Preservation

Three durable lessons from this feature warrant capture (triggered next):
1. **`PATTERN-CLIENT-SIDE-BULK-THUNK`** — mirror `bulkUpdateStatus` shape (SET_SAVING true → Promise.all(perSlug) → refetch → SET_SAVE_ERROR + re-throw → SET_SAVING false in finally) for every future bulk operation.
2. **`BUG-COMPOSER-AC-SLUG-DOUBLE-ENCODE`** — `encodeURIComponent(slug)` on the wpb-ac PUT path corrupts keys (composer sanitizer strips `%2F` rather than decoding). Always pass the slug raw; server-side `sanitize_ability_slug` (SEC-01) validates independently.
3. **`PATTERN-BULK-BUSY-OVERLAY-WP-NATIVE-SPINNER`** — reusable UI recipe: fullscreen backdrop-blur overlay + WP-native `.spinner is-active` scaled 48×48 + body scroll-lock via `useEffect`. Extracted as shared `BulkBusyOverlay` component per Constitution §VI DRY.
