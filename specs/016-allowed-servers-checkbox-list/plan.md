## Summary

Expose the `wpb-mcp-servers-list` REST endpoint and replace the free-text
`mcp_servers` input in AbilityForm with a dynamically populated checkbox list.
Two-file change: one `add_action` line in `includes/Main.php`, and the
replacement of a `<input type="text">` block in `AbilityForm.jsx`.

## Design

### PHP Change — `includes/Main.php`

Insert immediately after the existing `collect` registration block
(lines 282–283, `$mcp_servers_list` / `add_action rest_api_init collect 20`):

```php
// Register MCP servers REST endpoint (wpb-mcp-servers-list). Static callable
// — class string is valid for static methods per DEC-UTILITY-STATIC-ONLY.
$mcp_servers_rest = \WPBoilerplate\McpServersList\RestEndpoint::class;
$this->loader->add_action( 'rest_api_init', $mcp_servers_rest, 'register', 20 );
```

**Priority 20** — same as `collect`, fires after McpAdapter init at P15. Hook
registration order within the same priority = registration order in PHP; collect
runs first (registered first), then register. Servers are populated when the
endpoint builds its response.

**No new class/namespace required** — `RestEndpoint::register()` is a static
method. The Loader internally calls it as `[ClassName, 'register']` which PHP
routes to the static method correctly.

**PHPCS concern**: The `\WPBoilerplate\McpServersList\RestEndpoint::class`
expression is an FQN used only as a string (class constant). No `use` statement
needed. PHPCS will not flag it.

### JS Change — `AbilityForm.jsx`

#### State

Add two local state variables at the component level (no Redux):

```js
const [ mcpServers, setMcpServers ] = useState( null );    // null = loading
const [ mcpAdapterAvailable, setMcpAdapterAvailable ] = useState( true );
const [ mcpServersError, setMcpServersError ] = useState( false );
```

#### Fetch

```js
useEffect( () => {
    apiFetch( { path: '/wpb-mcp-servers-list/v1/servers' } )
        .then( ( data ) => {
            setMcpAdapterAvailable( data.adapter_available );
            setMcpServers( data.servers ?? [] );
        } )
        .catch( () => {
            setMcpServersError( true );
            setMcpServers( [] );
        } );
}, [] );
```

#### Toggle Handlers

```js
const handleServerToggle = ( serverId ) => {
    const current = draftAbility.mcp_servers ?? [];
    const next = current.includes( serverId )
        ? current.filter( ( id ) => id !== serverId )
        : [ ...current, serverId ];
    patch( { mcp_servers: next.length === 0 ? null : next } );
};

const handleAllServersToggle = () => {
    patch( { mcp_servers: null } );
};
```

> **Note**: `dispatch` from `useDispatch(STORE_NAME)` returns an action-creator object
> (e.g., `dispatch.updateDraft()`), NOT a raw Redux dispatch function. Always use the
> `patch()` helper (defined in AbilityForm.jsx via `useCallback`) to update draft fields.

#### Render Logic (replace the `<input type="text">` block)

```
if mcpServers === null → loading spinner / "Loading server list…"
else if mcpServersError → error notice + render stale checked items from draftAbility.mcp_servers
else if !mcpAdapterAvailable → "MCP adapter is not active." notice, no list
else if mcpServers.length === 0 → "No MCP servers registered yet." notice
else → render full checkbox list
```

Full list rendering:
1. "All servers (default)" checkbox — checked when `draftAbility.mcp_servers === null`
2. Union of `mcpServers` and stale IDs from `draftAbility.mcp_servers`
3. Each server: label = `server.name` (or raw ID for stale), sub-text = `server.id`
   (or "(not registered)" for stale)

#### Stale-ID Union

```js
const savedIds = Array.isArray( draftAbility.mcp_servers )
    ? draftAbility.mcp_servers : [];
const fetchedIds = ( mcpServers ?? [] ).map( ( s ) => s.id );
const staleIds = savedIds.filter( ( id ) => ! fetchedIds.includes( id ) );
const allItems = [
    ...( mcpServers ?? [] ),
    ...staleIds.map( ( id ) => ( { id, name: id, stale: true } ) ),
];
```

#### isNonDb Hint

Keep existing hint below the list:

{ isNonDb && (

### Null/Array Contract Invariants

| Condition | `mcp_servers` value |
|:---|:---|
| "All servers (default)" checked | `null` |
| One or more specific servers checked | `string[]` |
| Last specific server unchecked | `null` |
| `[]` empty array | NEVER — collapse to `null` immediately |
